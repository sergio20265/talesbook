# Nora Writer / Forest Writer — Android-приложение для написания книг

## Цель проекта

Создать красивое Android-приложение для личного письма: уютный, атмосферный редактор книг, глав и заметок, не похожий на сухой стандартный текстовый редактор.

Приложение должно ощущаться как личный магический кабинет автора: лес, свечи, тёплый свет, тёмно-зелёные оттенки, папоротники, книги, мягкая сказочная атмосфера, без перегруза и без “офисного” вида.

Основной пользователь — человек, который пишет художественные тексты, хочет вдохновения, уюта и визуальной красоты во время работы.

---

## Платформа

Android-приложение.

Предпочтительный стек:

* Kotlin
* Jetpack Compose
* Room для локальной базы данных
* DataStore для настроек
* MediaStore / SAF для выбора изображений и аудио
* Material 3, но с полностью кастомной атмосферной темой

Если проект создаётся на другом стеке, сохранить архитектуру и функциональные требования.

## Основные экраны

### 1. Стартовый экран / библиотека книг

Экран со списком книг.

Требования:

* карточки книг с обложкой
* название книги
* краткое описание
* дата последнего изменения
* кнопка добавления книги
* возможность редактировать и удалять книгу
* мягкий атмосферный фон
* пустое состояние: красивый экран “здесь начинается новая история”

Атмосфера:

* тёмный лесной фон
* глубокий зелёный / почти чёрный
* золотистые акценты
* мягкое свечение
* без ярких системных цветов

---

### 2. Экран книги

Показывает выбранную книгу и список глав.

Функции:

* редактировать название и описание книги
* менять обложку книги
* менять фон книги
* добавлять главы
* сортировать главы вручную
* открывать главу
* удалять / переименовывать главы
* показывать количество слов по книге

---

### 3. Экран редактора главы

Главный экран приложения.

Функции:

* редактирование названия главы
* большое поле текста
* автосохранение
* счётчик слов и символов
* режим фокуса без лишних кнопок
* возможность менять размер шрифта
* возможность менять цвет текста
* возможность менять фон редактора
* возможность прикреплять изображения
* возможность включать музыку фоном
* кнопка “назад” с сохранением
* мягкая анимация сохранения

Важно:

Редактор должен быть максимально спокойным. Никакого ощущения Google Docs или Word. Это должен быть личный сказочный письменный стол.

---

### 4. Экран оформления

Пользователь может настроить внешний вид приложения.

Функции:

* выбрать готовую тему
* загрузить свой фон приложения
* загрузить свой фон редактора
* выбрать цвет текста
* выбрать цвет акцента
* выбрать размер шрифта
* выбрать музыку фоном
* включить / выключить музыку
* настроить громкость

Готовые темы:

1. Evening Forest
   Тёмный лес, тёплый свет, свечи, папоротники.

2. Witch Library
   Книжные полки, гримуар, свечи, глубокий зелёный.

3. Moon Garden
   Ночная луна, мягкий синий, серебристые акценты.

4. Hobbit Room
   Тёплая нора, дерево, книги, уютный свет.

* сказочный лес

* ведьмовской дневник

## Хранение данных

Все данные должны храниться локально на устройстве.

Нужны:

* Room database для книг и глав
* DataStore для настроек
* URI выбранных изображений и аудио
* корректная работа с разрешениями Android

Автосохранение:

* при изменении текста
* debounce 500–1000ms
* при выходе с экрана
* при сворачивании приложения

## Важные UX-детали

* Приложение не должно выглядеть техническим.
* Все действия должны быть мягкими и понятными.
* Удаление — только с подтверждением.
* Автосохранение должно быть незаметным.
* Интерфейс должен помогать писать, а не отвлекать.
* Кнопки и панели должны быть минимальными.
* В редакторе важнее всего текст и атмосфера.

---

## Тон интерфейса

Тексты в интерфейсе должны быть мягкими:

Примеры:

* “Новая история”
* “Добавить главу”
* “Продолжить писать”
* “Тихое место для слов”
* “Здесь пока пусто, но история уже рядом”
* “Сохранено”
* “Музыка для письма”
* “Оформление пространства”

Не использовать сухие формулировки вроде:

* “Document”
* “Entity”
* “Record”
* “Input field”
* “Submit”

## Задача для Codex

Создай Android-проект с описанной архитектурой и MVP-функциональностью.

Начни с:

1. структуры проекта;
2. моделей данных;
3. Room database;
4. репозиториев;
5. ViewModel;
6. навигации;
7. экранов LibraryScreen, BookScreen, ChapterEditorScreen, AppearanceSettingsScreen;
8. базовой темы Evening Forest;
9. автосохранения текста главы;
10. выбора изображения для обложки книги и фона редактора.

Код должен быть аккуратным, расширяемым и пригодным для дальнейшего развития приложения.

Не делать временную заглушку вместо архитектуры. Лучше меньше функций, но качественно и красиво.
